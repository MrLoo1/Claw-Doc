# 第 6 章 多渠道接入

OpenClaw 的安全设计和最佳实践。

## 安全层级

```
┌─────────────────────────────────┐
│      网络安全 (HTTPS/WAF)        │
├─────────────────────────────────┤
│      访问控制 (IP/Token)         │
├─────────────────────────────────┤
│      执行沙箱 (Sandbox)          │
├─────────────────────────────────┤
│      数据加密 (Encryption)       │
└─────────────────────────────────┘
```

## API Key 安全

```yaml
# ❌ 错误：硬编码
models:
  providers:
    openai:
      apiKey: "sk-xxx"

# ✅ 正确：环境变量
models:
  providers:
    openai:
      apiKey: "${OPENAI_API_KEY}"
```

## 执行沙箱

敏感工具在沙箱中执行：
- 文件系统隔离
- 网络访问限制
- 进程资源限制

## 危险操作确认

以下操作需要用户确认：
- 删除文件
- 发送外部消息
- 执行系统命令

## 最佳实践

1. 使用 HTTPS
2. 定期更新依赖
3. 限制文件访问路径
4. 审计日志
5. 最小权限原则
