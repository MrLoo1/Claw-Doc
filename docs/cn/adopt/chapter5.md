# 第5章 记忆与上下文

OpenClaw 的记忆系统让助理能够记住重要信息。

## 记忆层级

```
┌─────────────────────────────────────┐
│         工作记忆 (Working)           │  ← 当前对话上下文
├─────────────────────────────────────┤
│         情景记忆 (Episodic)          │  ← 每日笔记
├─────────────────────────────────────┤
│         语义记忆 (Semantic)          │  ← 长期记忆
└─────────────────────────────────────┘
```

### 工作记忆

当前对话的消息历史，会话结束后清空。

### 情景记忆

存储在 `memory/YYYY-MM-DD.md`，记录每天的事项。

### 语义记忆

存储在 `MEMORY.md`，持久化的重要信息。

## 使用记忆

### 让助理记住

```
记住我的生日是3月15日
```

```
记住我喜欢喝拿铁，不要加糖
```

### 查询记忆

```
你还记得我的生日吗？
```

```
我跟你说过什么重要的事？
```

### 忘记信息

```
忘记我的生日信息
```

## 记忆文件

### MEMORY.md 示例

```markdown
# MEMORY.md

## 用户信息
- 姓名：Boss
- 生日：3月15日
- 时区：Asia/Shanghai

## 偏好
- 喜欢喝拿铁，不加糖
- 喜欢用中文交流

## 重要事项
- 每周五下午开例会
```

## 配置

```yaml
memory:
  enabled: true
  maxMessages: 50      # 工作记忆消息上限
  autoSave: true       # 自动保存到 MEMORY.md
```

## 隐私提示

- 记忆文件存储在本地，不会上传到云端
- API Key 等敏感信息不应写入记忆
- 可以随时编辑或删除记忆文件
